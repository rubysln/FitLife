## Эндпоинты

### Получение одного пользователя

**URL**: /api/users/{id}

**Метод**: GET

**Описание**: Этот эндпоинт позволяет получить информацию о пользователе по его идентификатору.

**Параметры запроса**:
- `id` (путь) - Идентификатор пользователя (тип Long).

**Пример запроса**:
``http
GET /api/users/123

{
  "id": 123,
  "name": "Имя пользователя",
  "email": "user@example.com"
}

Получение списка всех пользователей
URL: /api/users

Метод: GET

Описание: Этот эндпоинт возвращает список всех пользователей.

Пример запроса:
GET /api/users
Пример ответа:
[
  {
    "id": 1,
    "name": "Пользователь 1",
    "email": "user1@example.com"
  },
  {
    "id": 2,
    "name": "Пользователь 2",
    "email": "user2@example.com"
  }
  // Другие пользователи
]
Создание пользователя
URL: /api/users

Метод: POST

Описание: Этот эндпоинт позволяет создать нового пользователя.

Тело запроса:
{
  "name": "Имя пользователя",
  "email": "user@example.com",
}
Пример запроса:
POST /api/users
Пример ответа:
{
  "id": 123,
  "name": "Имя пользователя",
  "email": "user@example.com"
}
Удаление пользователя
URL: /api/users/{id}

Метод: DELETE

Описание: Этот эндпоинт удаляет пользователя по его идентификатору.

Параметры запроса:

id (путь) - Идентификатор пользователя (тип Long).
Пример запроса:
DELETE /api/users/123
Примечание: Этот эндпоинт не возвращает ответ в теле.

Обновление пользователя
URL: /api/users/{id}

Метод: PATCH

Описание: Этот эндпоинт позволяет обновить информацию о пользователе по его идентификатору.

Параметры запроса:

id (путь) - Идентификатор пользователя (тип Long).
Тело запроса:
{
  "name": "Новое имя пользователя",
}
Пример запроса:
PATCH /api/users/123
{
  "id": 123,
  "name": "Новое имя пользователя",
  "email": "user@example.com"
}

## PhysicalActivityDataController

### Получение данных по физической активности

**URL**: `/api/activities/{userId}/{activityId}`

**Метод**: GET

**Описание**: Этот эндпоинт позволяет получить данные по физической активности пользователя по его идентификатору пользователя (`userId`) и идентификатору активности (`activityId`).

**Параметры запроса**:
- `userId` (путь) - Идентификатор пользователя (тип Long).
- `activityId` (путь) - Идентификатор активности (тип Long).

**Пример запроса**:
``http
GET /api/activities/123/456
Пример ответа:

{
  "dataId": 456,
  "userId": 123,
  // Другие поля данных физической активности
}
Получение списка всех данных по физической активности для пользователя
URL: /api/activities/{userId}

Метод: GET

Описание: Этот эндпоинт возвращает список всех данных по физической активности для указанного пользователя.

Параметры запроса:

userId (путь) - Идентификатор пользователя (тип Long).
Пример запроса:

GET /api/activities/123
Пример ответа:

{
  "_embedded": {
    "activityList": [
      {
        "dataId": 456,
        "userId": 123,
        // Другие поля данных физической активности
      },
      {
        "dataId": 789,
        "userId": 123,
        // Другие поля данных физической активности
      }
      // Другие данные физической активности
    ]
  },
  "_links": {
    "self": {
      "href": "http://example.com/api/activities/123"
    }
  }
}
Создание данных по физической активности
URL: /api/activities/{userId}

Метод: POST

Описание: Этот эндпоинт позволяет создать новые данные по физической активности для пользователя.

Параметры запроса:

userId (путь) - Идентификатор пользователя (тип Long).
Тело запроса:

{
  "userId": 123,
  // Другие поля данных физической активности
}
Пример запроса:

POST /api/activities/123
Пример ответа:

{
  "dataId": 456,
  "userId": 123,
  // Другие поля данных физической активности
}
Обновление данных по физической активности
URL: /api/activities/{userId}/{activityId}

Метод: PATCH

Описание: Этот эндпоинт позволяет обновить данные по физической активности для пользователя по их идентификаторам (userId и activityId).

Параметры запроса:

userId (путь) - Идентификатор пользователя (тип Long).
activityId (путь) - Идентификатор активности (тип Long).
Тело запроса:

{
  "userId": 123,
  // Другие обновляемые поля данных физической активности
}
Пример запроса:

PATCH /api/activities/123/456
Пример ответа:

{
  "dataId": 456,
  "userId": 123,
  // Другие обновленные поля данных физической активности
}
Удаление данных по физической активности
URL: /api/activities/{activityId}

Метод: DELETE

Описание: Этот эндпоинт удаляет данные по физической активности по их идентификатору (activityId).

Параметры запроса:

activityId (путь) - Идентификатор активности (тип Long).
Пример запроса:

DELETE /api/activities/456
Примечание: Этот эндпоинт может вернуть 204 No Content в случае успешного удаления или 404 Not Found в случае отсутствия данных физической активности.
